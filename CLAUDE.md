# visit-local 프로젝트 안내

## 기본 정보

- 프로젝트명: visit-local
- 유형: 내 지역 관광 소개 정적 랜딩 페이지
- 작업 대상: 내 지역 관광/명소 소개 콘텐츠 및 방문 유도용 랜딩 화면
- 작업 기간: 필요 시 기재
- 목적: 방문자에게 내 지역 관광 정보를 빠르게 전달하고 관심을 유도하는 것

## 작업 범위

- 메인 랜딩 페이지 구성
- 관광 소개 문구, 섹션 구성, CTA 영역, 메뉴/탭/슬라이더 동작
- 이미지/아이콘/테마 스타일 유지
- 모바일 및 데스크톱에서의 기본 표시 동작 보장

## 제외 범위

- 백엔드 API 연동
- 실사용 회원/예약/로그인 기능
- 외부 CMS 또는 데이터베이스 연결

## 관련 파일

- `index.html` — 전체 페이지 구조와 스타일/스크립트가 포함된 메인 파일

## 구현 지침

- 페이지는 가볍고 자체 포함된 형태로 유지하세요.
- 특별한 요청이 없으면 외부 의존성을 추가하지 마세요.
- 큰 인라인 섹션을 수정할 때는 전체 재작성보다 타깃 기반 문자열 교체를 선호하세요.
- 기존 테마 구조와 `:root[data-theme="dark"]` 토글 동작은 유지하세요.

## 콘텐츠 규칙

- 공식 또는 기관 소스 이미지를 사용하세요.
- 가능하면 기존 SVG 심볼 아이콘 체계를 재사용하세요.
- 현재 랜딩 페이지의 구조와 말투를 유지하세요.

## Git 작업 규칙

- "커밋해줘" 요청이 있으면 현재 변경사항을 로컬 저장소에 한 번만 커밋합니다.
- "푸시해줘" 요청이 있으면 `origin main`과 `org main`에 모두 푸시합니다.
- "커밋하고 푸시해줘" 요청이 있으면 먼저 커밋하고, 그다음 `origin main`과 `org main`에 모두 푸시합니다.
- 강제 푸시는 하지 않습니다.
- `remote` 주소나 브랜치 추적 설정을 임의로 변경하지 않습니다.
- 다른 프로젝트 저장소는 건드리지 않습니다.
